1. How does the internet work?

    - Apa itu internet?

    Internet adalah Jaringan global komputer yang terhubung satu sama lain
    dan berkomunikasi melalui serangkaian protokol standar


    Internet adalah hasil eksperimen lain bernama ARPANET (Advanced Research Projects Agency Network)

    Vint Cerf

    1970

    Bob Kahn

      Pembuatan Protokol TCP/IP
      yaitu protokol yang digunakan untuk pertukaran data pada jaringan Internet.

      TCP (Transmission Control Protocoll) / IP (Internet Protocol)

      TCP / IP menjadi basis dari jaringan internet

    Ciri-ciri Internet

    • Internet terdiri dari sejumlah besar jaringan independen.
    • Tidak ada kendali pusat yang memutuskan bagaimana paket dirutekan
      atau bagaimana potongan jaringan dibentuk.
    • Operator-operator internet berkeinginan
      untuk memastikan bahwa ada sambungan ujung-ke-ujung dari setiap bagian jaringan.

    Bagaimana Informasi bergerak di internet?

      Permasalahan:

    • Kita harus mengirimkan bit-bit(atau informasi) melalui internet
      ke seluruh dunia, bukan hanya melalui satu tempat ke tempat lain
      dikota atau negara.

      • Penyelesaian:

    • Karena itu, kita membutuhkan metode pengiriman bit yang lebih cepat
      dan lebih stabil daripada menggunakan kawat ethernet.

      • Kesimpulan:

    • Kabel Serat Optik adalah salah satu cara untuk mengirimkan bit dengan
      kecepatan cahaya tanpa kehilangan sinyal dalam jarak yang sangat jauh
    • Kita dapat mengirimkan pancaran cahaya dari satu tempat ke tempat lain
      menggunakan kabel serat optik

2. What is HTTP?

    Pengertian HTTP

    • HTTP (Hypertext Transfer Protocol) adalah protokol komunikasi dasar yang digunakan untuk mengirimkan data di World Wide Web.
    • Ini adalah protokol berbasis teks yang memungkinkan pertukaran informasi antara client(web browser) dan server melalui permintaan(requests) dan respon(responses). HTTP berfungsi sebagai fondasi untuk semua komunikasi web, memungkinkan pengguna untuk mengambil halaman web, mengisi data formulir, mengunduh file, dan banyak lagi

    Siapa Penemu HTTP?

      Tim Berners-Lee

      HTTP diciptakan oleh Tim Berners-Lee pada akhir 1980-an dan awal 1990-an


      HTTP singkatan dari (HyperText Transfer Protocol)

      HTTP merupakan Intruksi khusus yang membantu internet tahu bagaimana
      mengirim dan menerima informasi antar perangkat!


      Contoh sederhana: ketika kita mengetik sebuah URL ataupun klik link
      Komputer kita akan mengirim permintaan ke server, kalau diibaratkan sepert ini:
      Hei, saya ingin informasi ini! Dan server menjawab dengan data yang ditunjuk menggunakan HTTP


      Metode dasar HTTP

      HTTP sangat penting karena memungkinkan kita membuat aplikasi web yang dapat diakses oleh berbagai jenis perankgat.


      Ada beberapa metode HTTP dasar, seperti:

      • GET: Digunakan untuk mengakses sebuah halaman web atau mendapatkan data dari server.
      • POST: Digunakan unntuk mengirimkan data ke server dan mmeprosesnya.
      • PUT: Digunakan untuk memperbarui data yang telah ada di serber.
      • DELETE: Digunakan untuk menghapus data yang telah ada di server.

      Status Code dan Header

      Status Code:
      HTTP memiliki beberapa kode status yang digunakan untuk menunjukkan hasil dari request.
      Beberapa contoh kode status adalah:

      • 200 OK: Menunjukkan bahwa request berhasil dilakukan.
      • 404 Not Found: Menunjukkan bahwa resource tidak ditemukan.
      • 500 Internal Server Error: Menunjukkan bahwa terjadi error pada server.

      Header:
      Header adalah bagian dari HTTP request atau response yang berisi informasi tambahan. Beberapa
      contoh header adalah:

      • Accept: Menunjukkan tipe data yang diinginkan oleh client (misalnya, JSON atau XML.)
      • Content-Type: Menunjukkan tipe data yang dikirimkan oleh server (misalnya, HTML atau JSON).
      • Authorization: Menunjukkan informasi autentikasi untuk request.

3. What is Domain Name?

    Pengertian Domain Name

    Domain Name adalah nama unik yang diberikan untuk sebuah website atau server i internet. Nama
    ini digunakan oleh pengguna untuk mengakses website atau server melalui alamat URL(Uniform Resource Locator).


    Contoh Domain Name adalah:


    • google.com
    • facebook.com
    • youtube.com

      • Pada dasarnya, Domain Name terdiri dari dua bagian:


      1. Nama Subdomain: Bagian awal dari Domain Name, sepeti "www" atau "blog".
      2. Nama TLD (Top-Level Domain): Bagian akhir adri Domain Name, seperti ".com", ".org", atau ".io".

      Domain Name sangat penting dalam pengembangan web karena memungkinkan pengguna untuk mengakses website atau server dengna lebih mudah dan lebih cepat.

4. What is Hosting

    Pengertian Hosting

    Hosting adalah layanan yang memungkinkan Anda untuk menampilkan website atau aplikasi web di internet. Dalam arti lain, Hosting adalah tempat dimana file-file website Anda disimpan dan dikirimkan ke pengguna melalui IP (Internet Protocol)


    Ada beberapa jenis Hosting, anata lain:

      1. Shared Hosting: Pada dasarnya, Shared Hosting adalah sebuah layanan yang memungkinkan Anda untuk berbagi ruang penyimpanan dengan pengguna lainnya. Harga terjangkaunya dan cocok untuk website kecil
      2. VPS (Virtual Private Server) Hosting: Pada dasarnya, VPS Hosting adalah sebuah layanan yang memberikan Anda sebuah virtual server pribadi di dalam sebuah fisik server. Kecilpun Anda memiliki kontrol penuh atas server Anda.
      3. Dedicated Hosting: Pada dasarnya, Dedicated Hosting adalah sebuah layanan yang memungkinkan Anda untuk memiliki sebuah server fisik yang ditujukan hanya untuk Anda sendiri. Mereka sangat cocok untuk website yang besar atau aplikasi web yang memerlukan kontrol penuh.

5. DNS and how it works?

    Pengertian DNS

    DNS (Domain Name System) adalah sebuah sistem yang memungkinkan Anda untuk menghubungkan nama domain ke alamat IP server.
    Dalam arti lain, DNS adalah sebuah direktori yang memungkinkan pengguna internet untuk menemukan website atau server dengan menggunakan nama domain yang unik.


    Berikut adalah bagaimana cara kerja DNS:


      1. Domain Name: Anda memiliki sebauh nama domain, seperti google.com
      2. DNS Server: Nama domain adna disimpan di dalam sebuah databse DNS Server
      3. Resolver: Pengguna internet memakai sebuah client, seperti browser, untuk menghubungkan ke nama domain anda.
      4. Root DNS Server: Client megnirim permintaan ke ROot DNS Server untu menemukan alamat IP server yang terkait dengan nama domain anda.
      5. Authoritative DNS Server: Root DNS SErver kemudian megnarahkan permintaan ke Authoritative DNS Server yang berada didalam zona DNS anda.
      6. Answer: Authoritative DNS Server kemudian memberikan jawaban, yaitu alamat IP server yang terkait dengan naam domain anda.

    DNS sangant penting dalam pengembangan web karena memungkinkan pengguna internet utuk mengakses website atau server dengan menggunakan nama domain yang unik.

6. Browsers and how they work?

    Bagaimana cara kerja Browsers?

    Browser adalah sebuah aplikasi yang memungkinkna pengguna untuk mengakses dan menggunakan internet.


      Berikut bagaimana cara keraja browser!

        1. User Interface: Browser memiliki sebuah antarmuka user yang memungkinkan pengguna untuk menginput alamat URL, mencari kata kunci, dan melakukan aksi lainnya.
        2. Rendering Engine: Browser memiliki sebuah engine rendering yang digunakan untuk menampilkan halaman web dengan cara yang benar.
        3. Navigation: Browser memungkinkan pengguna untuk berpindah antar halaman
        4. Cookies and Storage: Browser memungkinkan pengguna untuk menyimpan data seperti Cookies, Local Storage, dan Session Storage.
        5. Security: Browser memiliki sebuah fitur keamanan yang memungkinkan pengguna untuk melakukan verifikasi identitas, menahan cookie, dan melindungi data pengguna.

        Beberapa jenis browser umum digunakan antara lain:

          1. Google Chrome
          2. Mozilla Firefox
          3. Microsoft Edge
          4. Safari
